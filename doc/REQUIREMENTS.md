# Requirements Specification (요구사항 정의서)

## 1. 개요 (Overview)
본 문서는 모바일 기기를 활용하여 PC 화면을 제어하는 **'Mobile Laser Pointer'** 프로젝트의 비즈니스 및 사용자 요구사항을 정의한다.


## 2. 사용자 페르소나 (User Persona)
* **발표자 (Presenter):** 프레젠테이션 진행 중, 별도의 하드웨어 포인터 없이 자신의 스마트폰으로 화면을 가리키고 싶어 하는 사용자.


## 3. 핵심 요구사항 (Key Requirements)

### 3.1. 연결성 (Connectivity)
* **REQ-01:** 사용자는 별도의 복잡한 페어링 과정 없이 IP 주소 입력 또는 QR 코드 인식을 통해 모바일과 PC를 연결할 수 있어야 한다.
* **REQ-01-1:** PC 앱은 현재 IP 주소를 포함한 QR 코드를 생성하여 화면에 표시해야 한다.
* **REQ-01-2:** 모바일 앱은 QR 코드를 스캔하여 자동으로 IP 주소를 획득하고 연결을 시도해야 한다.
* **REQ-02:** 네트워크 연결은 프레젠테이션 환경(Wi-Fi) 내에서 끊김 없이 유지되어야 한다.
* **REQ-02-1:** PC 앱은 UDP 포트 50051에서 수신 대기 상태를 유지해야 한다.

### 3.2. 실시간성 (Real-time Performance)
* **REQ-03:** 모바일 기기의 움직임은 **지연(Latency) 없이** 즉각적으로 PC 화면에 반영되어야 한다. (레이저 포인터의 경험과 유사해야 함)
* **REQ-04:** 빠른 손떨림이나 급격한 이동에도 포인터가 부드럽게 움직여야 한다.

### 3.3. 사용성 (Usability)
* **REQ-05 (PC):** 포인터는 실행 중인 다른 프로그램(PPT, 웹브라우저 등)보다 **항상 위에(Always on Top)** 표시되어야 한다.
* **REQ-06 (PC):** 포인터 프로그램이 실행 중이라도, 마우스 클릭 등은 뒤에 있는 프로그램에 정상적으로 전달되어야 한다(Click-through). 배경 뒤의 프로그램은 마우스 이동, 클릭, 스크롤 등 모든 이벤트를 정상적으로 수신받을 수 있어야 한다.
* **REQ-07 (Mobile):** 사용자는 화면을 보지 않고도 포인터를 제어할 수 있어야 하므로, 직관적인 제스처나 파지법을 고려해야 한다.

## 4. 기능 요구사항 (Functional Requirements)

### 4.1. 모바일 앱 (Sender)
* **FUNC-REQ-01:** 자이로스코프 센서를 실시간으로 읽어 기기 회전 움직임을 감지한다.
* **FUNC-REQ-02:** 자이로스코프 데이터를 정규화된 좌표(-1.0 ~ 1.0)로 변환하여 포인터 위치를 업데이트한다.
* **FUNC-REQ-03:** UDP 프로토콜을 사용하여 실시간으로 좌표 데이터를 전송한다. 포트 번호는 50051을 사용한다.
* **FUNC-REQ-04:** IP 주소를 수동으로 입력하거나 QR 코드를 스캔하여 PC와 연결한다.
* **FUNC-REQ-05:** 연결 상태를 시각적으로 표시한다 (연결됨/끊김/연결 중).
* **FUNC-REQ-05-1:** Rsupport 디자인 시스템을 적용하여 직관적이고 전문적인 UI를 제공한다.

### 4.2. PC 앱 (Receiver)
* **FUNC-REQ-06:** 투명한 오버레이 윈도우를 생성하여 다른 앱 위에 포인터를 표시한다.
* **FUNC-REQ-07:** 윈도우를 항상 최상위(Always on Top)로 유지한다.
* **FUNC-REQ-08:** 마우스 이벤트를 투과시켜 뒤쪽 앱과의 상호작용을 허용한다.
* **FUNC-REQ-09:** UDP로 수신한 정규화된 좌표 데이터를 화면 좌표로 변환하여 Rsupport Orange 색상의 점으로 렌더링한다.
* **FUNC-REQ-10:** 현재 IP 주소를 포함한 QR 코드를 생성하여 화면에 표시한다.
* **FUNC-REQ-11:** 다중 모니터 환경에서 사용자가 대상 디스플레이를 선택할 수 있도록 한다.
* **FUNC-REQ-12:** 연결 상태를 자동으로 감지하고, 일정 시간 동안 패킷이 수신되지 않으면 연결 해제 상태로 전환한다.
* **FUNC-REQ-13:** Rsupport 디자인 시스템을 적용하여 전문적인 UI를 제공한다.

### 4.3. 연결 방식
* **FUNC-REQ-14:** 사용자는 IP 주소를 직접 입력하여 연결할 수 있다.
* **FUNC-REQ-15:** 사용자는 QR 코드를 스캔하여 자동으로 IP 주소를 획득하고 연결할 수 있다.
* **FUNC-REQ-16:** QR 코드 형식은 단순 IP 주소 문자열이다 (예: "192.168.0.1"). 포트 번호는 고정값(50051)을 사용한다.

## 5. 제약 사항 (Constraints)
* **플랫폼:** 
  * 모바일: Android, iOS 지원
  * PC: Windows, macOS 지원
  * 웹(Web) 플랫폼은 지원하지 않음
* **기술 스택:** Flutter 프레임워크를 기반으로 하며, 네이티브 코드는 필요한 경우(MethodChannel)에만 사용한다.
* **배터리:** 센서 사용으로 인한 과도한 배터리 소모를 지양해야 한다.
